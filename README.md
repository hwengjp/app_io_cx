## コンパイル手順

本アプリケーションは、TWELITE_Stageを使用してビルドおよび書き込みを行います。

### 必要な環境

- **TWELITE_Stage**: TWELITE開発環境（MWSTAGEに含まれる）
- **MWSDK**: Mono Wireless SDK（TWELITE SDK、MWSTAGEに含まれる）
- **TWELITE R3**: 書き込み対象のTWELITEボード

### ビルド&書き込み手順

1. デフォルト値設定ファイルの作成

`Common/config_private.h.template` から デフォルト値設定ファイル`Common/config_private.h`を作成してください。
詳細は[デフォルト設定値](#デフォルト設定値)のセクションを参照してください。

この設定ファイルで、アプリケーションID、論理デバイスID、チャネル、送信出力、オプションビットなどのデフォルト値を設定します。

2. **プロジェクトフォルダのコピー**

   TWELITE_Stageのワークスペースにプロジェクトフォルダをコピーします：

   ```
   <インストールフォルダ>\MWSTAGE\MWSDK\Wks_TweApps\
   ```

   このディレクトリに`App_IO_CX`フォルダをコピーしてください。

   **注意**: `App_IO_CX`フォルダの下位に同名フォルダ（`App_IO_CX/App_IO_CX/`）がありますが、**上位のフォルダ（`App_IO_CX/`）をコピー**してください。

3. **TWELITE R3の接続**

   TWELITE R3ボードをUSBケーブルでPCに接続します。

4. **TWELITE_Stageの起動**

   TWELITE_Stageアプリケーションを起動します。

5. **シリアルポートの選択**

   TWELITE_Stageのメニューから、接続されているTWELITE R3のシリアルポートを選択します：

   ```
   シリアルポート選択 → TWELITE R3 (<シリアル番号>)
   ```

6. **ファームウェアのビルド&書き込み**

   TWELITE_Stageのメニューから以下を選択します：

   ```
   アプリ書換 → TWELITE APPSビルド&書換 → App_IO_CX → App_IO_CX
   ```

   **書き込み成功のメッセージ**が表示されたら、書き込み完了です。

7. **EEPROMの消去**

   古い設定値をクリアする場合は、インタラクティブモードでEEPROMを消去します：

   ```
   インタラクティブモード → Extr Menu → EEPROM UTIL → Erase ALL sectors. → Reset System
   ```

   **注意**: EEPROMを消去すると、以前に保存されていた設定値がすべて削除され、デフォルト値に戻ります。

8. **設定値の変更（オプション）**

   インタラクティブモードを使用して、デフォルト設定値を変更できます。詳細は[デフォルト設定値](#デフォルト設定値)のセクションを参照してください。

## ハードウェア仕様

### 動作モード

M1ピンで動作モードを切り替えます（端子はプルアップなので、オープン=High、GND接続=Low）：

| M1 | 読み取り値 | モード | 説明 |
|----|----------|--------|------|
| オープン（O） | 1 (High) | `E_IO_MODE_PARNET` | 親機 |
| GND接続（G） | 0 (Low) | `E_IO_MODE_CHILD_CONT_TX` | 連射子機 |

**注意**: M2, M3ピンは使用されません。

**削除されたモード**:
以下の動作モードは削除されています：
- 中継機能（ROUTERモード）
- スリープモード（1秒/10秒間欠子機）
- 通常子機モード

### チャネル設定

EI1とEI2ピンの組み合わせでチャネルを設定します（端子はプルアップなので、オープン=High、GND接続=Low）：

| EI1 | EI2 | 読み取り値 | チャネル設定値 | 使用チャネル |
|-----|-----|----------|--------------|------------|
| オープン（O） | オープン（O） | 1, 1 | 0 | デフォルト/フラッシュ値（`CONFIG_DEFAULT_CHANNEL`） |
| GND接続（G） | オープン（O） | 0, 1 | 1 | ハードウェア設定（`CHANNEL_1`） |
| オープン（O） | GND接続（G） | 1, 0 | 2 | ハードウェア設定（`CHANNEL_2`） |
| GND接続（G） | GND接続（G） | 0, 0 | 3 | ハードウェア設定（`CHANNEL_3`） |

**注意**: 実際のチャネル設定は`config_private.h`の`CHANNEL_1`, `CHANNEL_2`, `CHANNEL_3`で変更できます。

**注意**: ウォッチドッグ出力機能が有効な場合、`PORT_EI2`は使用されず、`PORT_EI1`のみで制御（0または1の2通り）されます。

#### チャネル設定の選択方法

チャネル設定には、**ハードウェア設定（EI1/EI2ピン）**と**フラッシュメモリ設定（インタラクティブモードで設定した値）**の2つの方法があります。起動時にEI1/EI2ピンの状態を読み取り、自動的に選択されます（端子はプルアップなので、オープン=High、GND接続=Low）：

| EI1 | EI2 | 読み取り値 | 設定値 | 使用されるチャネル設定 |
|-----|-----|----------|--------|---------------------|
| オープン（O） | オープン（O） | 1, 1 | 0 | **フラッシュメモリの値**を使用<br>（インタラクティブモードで設定した値、または未設定の場合は`CONFIG_DEFAULT_CHANNEL`） |
| GND接続（G） | オープン（O） | 0, 1 | 1 | **ハードウェア設定**を使用（`CHANNEL_1`） |
| オープン（O） | GND接続（G） | 1, 0 | 2 | **ハードウェア設定**を使用（`CHANNEL_2`） |
| GND接続（G） | GND接続（G） | 0, 0 | 3 | **ハードウェア設定**を使用（`CHANNEL_3`） |

**注意**: 実際のチャネル設定は`config_private.h`の`CHANNEL_1`, `CHANNEL_2`, `CHANNEL_3`で変更できます。

**動作の詳細**:

1. **設定値0（EI1=オープン, EI2=オープン、読み取り値=1,1）の場合**:
   - フラッシュメモリに保存されているチャネル設定（インタラクティブモードで設定した値）を使用
   - フラッシュメモリに値が保存されていない場合、`CONFIG_DEFAULT_CHANNEL`（`config_private.h`で定義）が使用される
   - インタラクティブモードで設定したチャネル値が優先される

2. **設定値1, 2, 3（EI1/EI2ピンで指定、GND接続=Low、オープン=High）の場合**:
   - ハードウェア設定（`CHANNEL_1`, `CHANNEL_2`, `CHANNEL_3`）が使用される
   - フラッシュメモリの値は無視される
   - 起動時にEI1/EI2ピンの状態を読み取り、対応するチャネルに自動設定される

3. **オプションビットによる制御**:
   - `E_APPCONF_OPT_NO_C1C2_CONFIG`（0x0200）が有効な場合、ハードウェア設定（EI1/EI2ピン）は無視され、常にフラッシュメモリの値が使用される
   - このオプションを有効にすると、EI1/EI2ピンの状態に関係なく、フラッシュメモリに保存されているチャネル値（インタラクティブモードで設定した値）が使用される

### ポート設定

入出力ポートの割り当ては、オプションビット `E_APPCONF_OPT_PORT_TBL1` (0x1000) と `E_APPCONF_OPT_PORT_TBL2` (0x2000) の組み合わせで選択できます。

| PORT_TBL1 | PORT_TBL2 | テーブル番号 | 説明 |
|-----------|-----------|------------|------|
| 0 | 0 | 0 (SET0) | 子機入力12（親機出力12） |
| 1 | 0 | 1 (SET1) | 子機入力8・出力4（親機出力8・入力4） |
| 0 | 1 | 2 (SET2) | 子機入力6・出力6（親機出力6・入力6） |
| 1 | 1 | 3 (SET3) | 子機出力12（親機入力12） |

**詳細**: ポートテーブルの詳細なピン割り当てについては、[App_IO ピン割り当て](https://mono-wireless.com/jp/products/TWE-APPS/App_IO/pinassign.html)を参照してください。

## デフォルト設定値

本アプリケーションのデフォルト設定値は、`Common/config_private.h`で定義されています。インタラクティブモードで表示されるデフォルト値は、このファイルを編集することで変更できます。

**初回セットアップ**: `Common/config_private.h.template`を元に`Common/config_private.h`を作成してください。

### 設定項目とデフォルト値

| 設定項目 | デフォルト値（`config_private.h`） | 説明 |
|---------|------------------------------|------|
| アプリケーションID | `CONFIG_DEFAULT_APPID` | ネットワークを論理的に分離するID（32bit） |
| 論理デバイスID | `CONFIG_DEFAULT_LOGICALID` | デバイス識別ID（1-100など） |
| チャネル（デフォルト） | `CHANNEL_DEFAULT` | ハードウェア設定用デフォルトチャネル（EI1=O, EI2=O で使用） |
| チャネル（設定値1） | `CHANNEL_1` | EI1=G, EI2=O で使用されるチャネル |
| チャネル（設定値2） | `CHANNEL_2` | EI1=O, EI2=G で使用されるチャネル |
| チャネル（設定値3） | `CHANNEL_3` | EI1=G, EI2=G で使用されるチャネル |
| 送信出力と再送回数 | `CONFIG_DEFAULT_POWER_N_RETRY` | 再送: 9回=最大, 出力: レベル3=最大 |
| 子機連続0.03秒モード(3)のサイクル | `CONFIG_DEFAULT_FPS` | 毎秒送信回数（4, 8, 16, 32のいずれか） |
| 暗号化 | `CONFIG_DEFAULT_ENCENABLE` | 暗号化の有効/無効（0:無効, 1:有効） |
| 暗号鍵 | `CONFIG_DEFAULT_ENCKEY` | 暗号鍵（15文字） |
| オプションビット | `CONFIG_DEFAULT_OPTBITS` | 各ビットに紐付いた設定（32bit） |

### アプリケーションIDの推奨範囲

アプリケーションID（`CONFIG_DEFAULT_APPID`）は、ネットワークを論理的に分離するための32bitのIDです。

モノワイヤレスは、ユーザーが独自に使用するApp IDとして以下の範囲を推奨しています：

- **推奨範囲**: `0x80000000` ～ `0xFFFFFFFF`

この範囲は、モノワイヤレスが使用していない領域のため、他のアプリケーションとバッティングしにくくなります。

**注意**: 
- 先頭や末尾が`0000`や`FFFF`の値は無効です
- 同じネットワーク内のすべてのデバイスに同一のApp IDを設定する必要があります

### 論理デバイスIDの詳細

論理デバイスID（`CONFIG_DEFAULT_LOGICALID`）は、ネットワーク内でデバイスを識別するための論理アドレスです。同じネットワーク内で複数の子機を使用する場合に、各子機に異なるIDを設定することで識別できます。

#### 論理アドレスの値と意味

| 値 | 意味 | 説明 |
|----|------|------|
| `0` | 親機 | ネットワークの親機（`LOGICAL_ID_PARENT`） |
| `1-127` | 子機 | 個別の子機を識別するID（`IS_LOGICAL_ID_CHILD`で判定） |
| `120` | 子機全般 | すべての子機宛てのブロードキャスト（`LOGICAL_ID_CHILDREN`） |
| `240` | ペアリング | ペアリングモード用（`LOGICAL_ID_PAIRING`） |
| `254` | リピータ | 中継機用（`LOGICAL_ID_REPEATER`、本アプリでは未使用） |
| `255` | 無効/ブロードキャスト | 通信しない、または全デバイス宛て（`LOGICAL_ID_BROADCAST`） |

#### 使用例

- **親機**: `LOGICALID = 0`（固定）
- **子機1**: `LOGICALID = 1`
- **子機2**: `LOGICALID = 2`
- **子機3**: `LOGICALID = 3`
- ...

#### 注意事項

- 親機は常に`0`を使用します（M1=オープン（High）で親機モードに設定した場合、自動的に`0`が設定されます）
- 子機は`1-127`の範囲で設定します（デフォルトは`1`）
- 同じネットワーク内で、複数の子機に同じIDを設定しないでください
- インタラクティブモードで設定値を変更するか、カスタムスロットを使用して複数の子機に異なるIDを設定できます

### チャネルの詳細

チャネル設定は、無線通信で使用する周波数チャネルを指定する設定です。物理的にネットワークを分離するために使用されます。

#### チャネルの範囲

- **有効範囲**: `11` ～ `26`
- **デフォルト値**: `CONFIG_DEFAULT_CHANNEL`（`config_private.h`で定義）

#### 混信を避けるための推奨事項

標準アプリ（App_Tweliteなど）との混信を避けるため、以下の点に注意してください：

1. **標準アプリのデフォルト値の回避**
   - 標準アプリのデフォルトチャネル（通常は`18`）は多くのデバイスで使用される可能性が高いため、**避けることを推奨**します
   - 本アプリケーションのデフォルト値（`CONFIG_DEFAULT_CHANNEL`）も、標準アプリと同じ値に設定しないことを推奨します

2. **推奨チャネル範囲**
   - **推奨**: 標準アプリでよく使用されるチャネル（通常は`11`, `12`, `18`, `21`, `25`など）を避け、使用頻度の低いチャネルを選択
   - 特に、標準アプリのデフォルトチャネルや、よく使用されるチャネルから離れた位置を選択することを推奨します
   - 本アプリケーションのハードウェア設定（`CHANNEL_1`, `CHANNEL_2`, `CHANNEL_3`）も、標準アプリでよく使用されるチャネルと重複しないように設定することを推奨します

3. **チャネル選択の目安**
   - 同一環境で標準アプリと本アプリケーションを併用する場合、標準アプリで使用されるチャネルとは異なるチャネルを割り当ててください
   - 標準アプリでよく使用されるチャネル（`11`, `12`, `18`, `21`, `25`など）を避け、使用頻度の低いチャネル（例：`13`, `14`, `15`, `19`, `20`, `22`, `23`, `24`, `26`）を選択することを推奨します
   - チャネル設定は`config_private.h`で変更できます

4. **チャネルアジリティ機能**
   - チャネルアジリティ機能を使用すると、最大3チャネルを指定してチャネルアジリティ機能を有効化できます
   - この機能により、指定した複数のチャネル間で自動的に切り替え、混信に対する耐性が向上します
   - **重要**: **低レイテンシーが要件の場合は、チャネルアジリティ機能を使用しないでください**。チャネル切り替えによる遅延が発生する可能性があります

5. **Wi-Fiとの干渉を避けるチャネル選択**
   
   TWELITEは2.4GHz帯を使用しており、Wi-Fi（IEEE 802.11b/g/n）と同じ周波数帯域のため、干渉が発生する可能性があります。
   
   **Wi-Fiと干渉しにくいチャネル選択肢**:
   
   - **推奨チャネル**: `13`, `14`, `15`, `19`, `20`, `22`, `23`, `24`, `26`
   - **特に推奨**: `13`, `14`, `15`, `19`, `20`, `22`, `23`, `24`, `26`（Wi-Fiの主要チャネル1、6、11の周波数帯から離れた位置）
   
   **避けるべきチャネル**:
   
   - `11`, `12`: Wi-Fiのチャネル11（2462 MHz）の周波数帯と近接している可能性
   - 標準アプリのデフォルトチャネル（通常は`18`）: 多くの標準アプリデバイスで使用される可能性が高い
   - 標準アプリでよく使用されるチャネル（`11`, `12`, `18`, `21`, `25`など）: 標準アプリとの混信リスク
   
   **選択のポイント**:
   
   - Wi-Fiルーターがチャネル1、6、11を使用している場合、これらの周波数帯から離れたTWELITEチャネルを選択してください
   - 標準アプリでよく使用されるチャネルを避け、使用頻度の低いチャネルを選択してください
   - 周波数帯の端の方（チャネル26付近など）は、Wi-Fiの主要チャネルから離れているため、干渉が少ない傾向があります
   - 実際の使用環境でテストを行い、最適なチャネルを選択することを推奨します
   - Wi-Fiアナライザーアプリなどを使用して、周囲のWi-Fiネットワークのチャネル使用状況を確認し、干渉の少ないチャネルを選択すると効果的です

#### 注意事項

- 同じネットワーク内のすべてのデバイス（親機・子機）に**同一のチャネル**を設定する必要があります
- チャネルが異なるデバイス同士は通信できません
- チャネル設定は、EI1/EI2ピンによるハードウェア設定、またはインタラクティブモードでの設定変更が可能です
- **低レイテンシー要件の場合**: チャネルアジリティ機能は使用せず、単一チャネルを使用してください
- **チャネル設定の優先順位**: EI1/EI2ピンが設定値1, 2, 3の場合、ハードウェア設定が優先されます。設定値0（両方オープン）の場合のみ、フラッシュメモリの値（インタラクティブモードで設定した値）が使用されます

### 送信出力と再送回数の詳細

送信出力と再送回数（`CONFIG_DEFAULT_POWER_N_RETRY`）は、XY形式の2桁16進数で指定します：

- **X桁（上位4bit）**: 再送回数
  - `0`: デフォルト2回
  - `1-9`: 指定回数
  - `F`: 無効
- **Y桁（下位4bit）**: 送信出力レベル
  - `3`: 最大
  - `2`: 中
  - `1`: 小
  - `0`: 最小

例：`0x42` = 再送4回、出力レベル2（中）

### オプションビットの詳細

オプションビット（`CONFIG_DEFAULT_OPTBITS`）は、32bitのビットフラグで、アプリケーションの動作を制御する各種オプションを有効/無効にする設定です。各ビットが特定の機能に対応しており、複数のオプションを同時に有効にすることができます。

#### オプションビット一覧

| ビット位置 | 値（16進数） | オプション名 | 説明 |
|-----------|------------|------------|------|
| Bit 0 | `0x0001` | `E_APPCONF_OPT_LOW_LATENCY_INPUT` | Hi→Lo検知後すぐに送信 |
| Bit 1 | `0x0002` | `E_APPCONF_OPT_LOW_LATENCY_INPUT_SLEEP_TX_BY_INT` | スリープ時、H→L検出で割り込み要因ポートのみ送信 |
| Bit 4 | `0x0010` | `E_APPCONF_OPT_ACK_MODE` | ACK付き通信を行う |
| Bit 5 | `0x0020` | `E_APPCONF_OPT_NO_REGULAR_TX` | REGULAR通信（定期送信）を無効にする |
| Bit 8 | `0x0100` | `E_APPCONF_OPT_ON_PRESS_TRANSMIT` | 押し下げ時のみ送信する特殊動作モード（リモコンモード） |
| Bit 9 | `0x0200` | `E_APPCONF_OPT_NO_C1C2_CONFIG` | C1/C2ポートによる設定切り替えを無効にする |
| Bit 10 | `0x0400` | `E_APPCONF_OPT_DI_INVERT` | DIO入力を反転する |
| Bit 11 | `0x0800` | `E_APPCONF_OPT_NO_PULLUP_FOR_INPUT` | DI入力にプルアップを適用しない |
| Bit 12 | `0x1000` | `E_APPCONF_OPT_PORT_TBL1` | ポート割り当てテーブル選択（Bit 0） |
| Bit 13 | `0x2000` | `E_APPCONF_OPT_PORT_TBL2` | ポート割り当てテーブル選択（Bit 1） |
| Bit 16 | `0x10000` | `E_APPCONF_OPT_CHILD_RECV_OTHER_NODES` | 子機通常モードで受信を可能とする |
| Bit 17 | `0x20000` | `E_APPCONF_OPT_CHILD_RECV_NO_IO_DATA` | IO状態のUART出力を抑制する |
| Bit 18 | `0x40000` | `E_APPCONF_OPT_WATCHDOG_OUTPUT` | PORT_EI2をウォッチドッグ出力とする |
| Bit 22 | `0x400000` | `E_APPCONF_OPT_DO_INVERT` | DIO出力を反転する |

#### 主なオプションの詳細

##### ACK付き通信 (`0x0010`)

ACK（Acknowledgment）付き通信を有効にします。子機が親機に送信する際、親機からの応答を待つことで通信の確実性が向上します。

- **有効時**: 子機から親機への送信でACKを要求
- **無効時**: ACKなしで送信（デフォルト）

##### REGULAR通信の無効化 (`0x0020`)

定期送信（REGULAR通信）を無効にします。IO変化時のみ送信する動作になります。

- **有効時**: 定期送信を無効化
- **無効時**: IO変化時と定期送信の両方で送信（デフォルト）

##### リモコンモード (`0x0100`)

押し下げ時のみ送信する特殊動作モードです。ボタンを押している間は送信せず、押し下げた瞬間のみ送信します。

- **有効時**: 押し下げ時のみ送信
- **無効時**: IO変化時に送信（デフォルト）

##### ウォッチドッグ出力 (`0x40000`)

PORT_EI2をウォッチドッグ出力として使用します。有効にすると、PORT_EI2からパルスが出力され、正常動作を監視できます。

- **有効時**: PORT_EI2からウォッチドッグパルスを出力
- **無効時**: PORT_EI2は通常のIOとして使用（デフォルト）

**注意**: ウォッチドッグ出力が有効な場合、PORT_EI2はチャネル設定入力に使用できません。

##### 子機での他ノード受信 (`0x10000`)

子機が他のノード（親機以外の子機など）からのパケットを受信できるようにします。

- **有効時**: 子機が他のノードからのパケットを受信可能になる（受信モードが常時有効）
- **無効時**: 子機は親機からのパケットのみを受信（デフォルト）

**動作の詳細**:
- このオプションを有効にすると、子機は`RX_STATE_BOOT_ON | RX_STATE_CONTINUOUS`モードに設定され、常時受信可能になります
- 出力ポート数が0より大きい場合も、自動的に受信可能モードになります（他のノードからの制御を受ける必要があるため）
- ただし、IO状態を実際に反映するのは親機⇔子機間の通信のみです
- 他の子機からのパケットを受信できますが、IO状態の設定には使用されません

**使用例**:
- 複数の子機間で情報を共有したい場合
- デバッグや監視目的で、子機が他のノードの通信を監視したい場合

##### IO状態のUART出力抑制 (`0x20000`)

子機が受信したIOデータのUART出力を抑制します。

- **有効時**: 子機が受信したIOデータのUART出力を抑制（シリアルポートに出力しない）
- **無効時**: 子機が受信したIOデータをUART出力する（デフォルト）

**動作の詳細**:
- このオプションは、子機が親機から受信したIOデータをシリアルポート（UART）経由で出力するかどうかを制御します
- 条件：`bSetIo`がTRUE（親機⇔子機間の通信でIO状態が設定される場合）かつ、VERBOSEモードでない場合にのみ影響します
- VERBOSEモードの場合は、このオプションの設定に関係なく出力されます
- UART出力は`SERCMD_ID_INFORM_IO_DATA`コマンド形式で、ASCII形式で出力されます

**使用例**:
- シリアルポートの出力を減らして、通信量を削減したい場合
- デバッグ出力を抑制して、必要な情報のみを出力したい場合
- 複数の子機が動作している環境で、UART出力による混雑を避けたい場合

##### 入出力の反転

- **`E_APPCONF_OPT_DI_INVERT` (`0x0400`)**: DIO入力を反転（Hi/Loが逆になる）
- **`E_APPCONF_OPT_DO_INVERT` (`0x400000`)**: DIO出力を反転（Hi/Loが逆になる）

#### オプションビットの設定方法

オプションビットは、複数のオプションを同時に有効にする場合、各ビットの値を**OR演算**で組み合わせます。

**例**:
- ACK付き通信のみ有効: `0x0010`
- ACK付き通信 + リモコンモード: `0x0010 | 0x0100 = 0x0110`
- ウォッチドッグ出力 + 入力反転: `0x40000 | 0x0400 = 0x40400`

- `Common/config_private.h`は`.gitignore`に登録されているため、Gitにチェックインされません
- テンプレートファイル（`Common/config_private.h.template`）はGitに含まれています
- 設定値を変更した後は、再コンパイルが必要です
- 複数の開発者が作業する場合、各自が`config_private.h`を作成する必要があります
